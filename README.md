# Video Translate Transcriber website

The public website and launch workspace for Video Translate Transcriber, built with Astro. Primary calls to action open the browser-based project setup at `/app`; the iOS processing and billing implementation remain separate.

## Run it locally

```sh
npm install
npm run dev
```

Open the local address Astro prints. Build and validate the production version with:

```sh
npm run check
npm run build
npm run validate
```

`dist/` is generated by the build and is intentionally ignored by Git.

## Source of truth for public facts

Keep real public facts in [src/data/site.ts](src/data/site.ts). It contains the site name, app path, page metadata, languages, visible access options, crawler policy, verification fields, and all unresolved launch values. The pre-launch canonical domain uses the reserved `.invalid` suffix on purpose; replace it with the real HTTPS domain before deployment.

Do not publish a public App Store link, support address, legal entity, privacy date, analytics configuration, verification token, or social account until it is verified. The privacy and terms pages are currently `noindex` placeholders, rather than final legal policies.

## Discovery features

- Canonical URLs, page-specific descriptions, Open Graph tags, and Twitter card metadata are emitted by [src/layouts/BaseLayout.astro](src/layouts/BaseLayout.astro).
- [src/pages/robots.txt.ts](src/pages/robots.txt.ts) generates crawler policy from `site.ts`.
- `@astrojs/sitemap` generates `sitemap-index.xml` from the same canonical configuration at build time.
- [src/pages/llms.txt.ts](src/pages/llms.txt.ts) and [src/pages/llms-full.txt.ts](src/pages/llms-full.txt.ts) provide supplemental AI-readable references.
- [src/data/structured-data.ts](src/data/structured-data.ts) emits WebSite, WebPage, BreadcrumbList, FAQPage, and MobileApplication data only where its content is visible on the page.

## Optional integrations

Analytics is intentionally disabled in `site.ts`. Add it only after choosing a provider and confirming the privacy treatment.

IndexNow is also disabled. After deployment, place the provider key file at the production host, set `INDEXNOW_ENABLED=true`, `INDEXNOW_HOST`, and `INDEXNOW_KEY`, then run:

```sh
npm run indexnow -- https://your-domain.example/changed-page
```
